*Чому секрети так потребують RBAC*

Секрети K8s розроблені для зберігання секретів. Як ви бачите, вирішуючи виклик, секрети вже кодуються в `base64`.

Секрети не повинні, якщо вони не зашифровані, закомічуватися в Git. Якщо вони є, секрет залишається в історії Git, навіть якщо ви його видалите. Завжди використовуйте RBAC для обмеження доступу до секретів K8s.
