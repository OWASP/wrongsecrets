Ви можете вирішити цей виклик такими альтернативними способами:

1. Перевірте стан configmap у Git:
 - Знайдіть, де в Git ми зберегли `secrets-config.yml`.
 - Перегляньте поле `Data`: що ви там можете знайти?

2. Запитайте ввічливо за допомогою Kubectl:
 - Виконайте `kubectl get configmap -n default`, а потім `kubectl describe configmap <ім'я>`.
