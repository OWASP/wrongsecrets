*Чому публікація скріншотів журналів у issue GitHub є поганою ідеєю*

Не слід публікувати скріншоти журналів у issue GitHub, тому що:
1. Дані є публічними (навіть у приватному репозиторії, часто більш публічними, ніж ви сподіваєтеся)
2. Скріншоти не можна редагувати як текст. Навіть коли секрет розмитий, це може бути оборотним.

Перевіряйте на витоки секретів у виводі та використовуйте блоки коду markdown для публікації журналів у issue GitHub.
