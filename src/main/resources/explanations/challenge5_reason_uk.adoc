*Чому використання configmap не є хорошою ідеєю*

ConfigMap ніколи не були розроблені для зберігання секретів. Вони були розроблені для зберігання додаткових даних для вашого Job, Pod та інших ресурсів K8s.

ConfigMap часто доступні для читання всіма, хто потребує розробляти/обслуговувати процеси, що вимагають даних із configmap. Без обмежень RBAC configmap є небезпечними сховищами для секретів.
